## How to Contribute

### Adding a Resource

1. Fork the repository
2. Add your resource in the appropriate category
3. Follow the format:
   ```markdown
   - [Resource Name](URL) - Brief description
   ```
4. Ensure the resource is:
   - Relevant to application security
   - High quality and useful
   - Properly categorized
   - Not a duplicate
5. Create a pull request

### Guidelines

- **Quality over quantity**: Only add resources you've personally used or verified
- **Keep descriptions concise**: 1-2 sentences maximum
- **Maintain alphabetical order**: Within each category
- **No self-promotion**: Unless genuinely valuable to the community
- **Check for duplicates**: Search before adding
- **Test links**: Ensure all links work

### Categories

Add resources to the most appropriate category:
- Learning Resources
- Tools
- Books
- YouTube Channels
- Blogs
- Platforms
- etc.

### Pull Request Process

1. Update README.md with your addition
2. Ensure your PR description explains what you're adding and why
3. One resource per PR (makes review easier)
4. Be patient - reviews may take a few days
